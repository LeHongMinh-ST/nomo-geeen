R2-01 · Tenant login and identity done

decision

Each user currently belongs to exactly one tenant. Login resolves that tenant from a unique username/email/phone + password match. The client no longer sends a tenant code; ambiguous duplicate matches fail generically. The response still returns tenant metadata and current role grants. Multi-tenant membership and tenant selection are deferred.

verification

Tenant-auth unit 4/4, auth + provisioning regression 69/69, AppModule smoke 1/1, backend build and Biome checks pass.
