R1-01 · Tenant owner creation API

File: tasks/task-R1-01-tenant-owner-creation-api.md · Requirements R1, R2
done
risk2026-07-19

Latent adapter-pg P2002 bug found & fixed. Under @prisma/adapter-pg, a P2002 error carries no meta.target. The original mapCreateError read only meta.target, so every unique violation defaulted to SLUG_TAKEN — USERNAME_TAKEN was unreachable and unrelated violations were mislabeled. The real fields live at meta.driverAdapterError.cause.constraint.fields (+ originalMessage). New uniqueViolationFields() reads both the standard and adapter locations to stay adapter-agnostic.

decision2026-07-19

Provisioning is a single prisma.$transaction: seed 3 per-tenant roles (OWNER/MANAGER/STAFF, grants cloned from system templates) → create tenant (mode:SIMPLE, status:ACTIVE, seatBonus default 10) → create OWNER user bound to the seeded OWNER role → two in-tx audit rows (TENANT_CREATE, USER_CREATE) via AuditLogger.writeInTx. Password hashed outside the tx; owner XOR password enforced in the service (PASSWORD_MODE_INVALID).

codebase-reality2026-07-19

Rollback cannot be triggered by spying prisma.user.create — the tx client is a distinct proxy. The reliable trigger is jest.spyOn(audit, 'writeInTx') (the shared singleton called inside the tx): first call resolves (real TENANT_CREATE row), second rejects → whole tx rolls back. Global ValidationPipe (whitelist + forbidNonWhitelisted + transform) means nested DTOs need @IsDefined()+@IsObject()+@ValidateNested()+@Type(); bare @ValidateNested() does not reject undefined (M1 fix).

verification2026-07-19

pnpm build (exit 0); pnpm test:e2e -- tenants-create = 7/7 passed (403 authz, 201 generated pwd, 201 explicit pwd, 409 SLUG_TAKEN, 400 missing-owner regression, 400 PASSWORD_MODE_INVALID, in-tx rollback → 0 orphan rows); pnpm test -- tenants.service = 16/16 passed (incl. deterministic USERNAME_TAKEN/SLUG_TAKEN mapping via real adapter-pg P2002 payload). Biome clean on scope. code-auditor: 2 MAJOR (M1 missing-owner→500, M2 USERNAME_TAKEN unproven) resolved; m3/m4/m5 minors deferred as tech debt.

R2-01 · Tenant users management API

File: tasks/task-R2-01-tenant-users-management-api.md · Requirements R3, R8
done
decision2026-07-20

New TenantUsersController/TenantUsersService under admin/tenants/:tenantId/users, gated admin.tenant-user:view (GET) / admin.tenant-user:manage (mutations). Seat rule: effectiveMaxUsers = (activePlan.maxUsers ?? 0) + tenant.seatBonus, plan from a subscription in ACTIVE/TRIALING within window; activeCount = users status=ACTIVE, deletedAt:null (DISABLED/INVITED do not consume seats). Create + reactivate re-read seat inside a serializable $transaction; last-owner guard counts active OWNERs in the same tx before demote/deactivate. A plan-less tenant seats purely on seatBonus (planCode:null).

codebase-reality2026-07-20

Cross-tenant access uses findFirst({id, tenantId, deletedAt:null}) → 404 (never 403, no existence leak). PATCH edit is a strict @Exclude()/@Expose() whitelist (fullName/username/phone/email); non-whitelisted fields are rejected upstream by the global forbidNonWhitelisted pipe (400) — role change is a separate PATCH :userId/role endpoint. Reset-password forces mustChangePassword=true; password/reset both enforce a {password} XOR {generatePassword} discriminated union. USER_CREATE audit shares the create tx via writeInTx; the P2002 → USERNAME_TAKEN mapper reads both standard and adapter-pg nested shapes.

risk2026-07-20

Audit spec-gap (carried forward): only USER_CREATE is recorded. Edit/deactivate/reactivate/changeRole/reset-password write no audit row — the AuditAction enum has no USER_UPDATE/DEACTIVATE/REACTIVATE/RESET_PASSWORD and R0-01 forbids inventing enum values. Needs a follow-up enum migration if per-mutation audit is later required. Audit fix from review: seat + last-owner counts now include deletedAt:null, closing a latent security-count drift ahead of any future user soft-delete.

verification2026-07-20

Build exit 0 · unit 18/18 (tenant-users.service.spec.ts) · e2e 8/8 (test/tenant-users.e2e-spec.ts: 403 gate, list+SeatUsage no passwordHash, create, seat-full 409, last-owner 409, cross-tenant 404, non-whitelist 400, reset forces mustChangePassword) · biome clean (0 contributed errors) · code-auditor zero CRITICAL (M1 + m1 fixed).
